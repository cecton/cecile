When writing git commit messages, follow Conventional Commits format:

    <type>: <description>

Types: feat, fix, chore, docs, style, refactor, test, ci, perf, build

Rules:
1. Subject line (first line) must be 79 characters or fewer
2. Use ASCII characters only — no emoji, no Unicode symbols
3. Use lowercase for type and description
4. No period at the end of the subject line
